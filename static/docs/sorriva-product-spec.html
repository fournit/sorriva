Zones & Audio

◎ Planned
Sorriva discovers every audio device on your network — Sonos, Bluesound, HomePod, and any AirPlay 2-compatible hardware — and presents them as a single unified zone list. No protocol awareness needed.
Discovered via
AirPlay 2 (mDNS/Bonjour) Sonos local API Bluesound BluOS
Play to multiple rooms simultaneously
◎ Planned
Group any combination of Sonos zones via the Sonos API, or any combination of Bluesound zones via the BluOS API. Sorriva creates and recalls these groups — the hardware handles sync natively at firmware level, the same way Roon does it. Cross-ecosystem groups (Sonos + Bluesound together) use AirPlay 2 for sync.
Delivery by group type
Sonos-only group → Sonos API → 24-bit/48kHz lossless Bluesound-only group → BluOS API → 24-bit/192kHz lossless Mixed hardware group → AirPlay 2 → 16-bit/44.1kHz lossless CD

Same-ecosystem groups get native API quality. Mixed groups get AirPlay 2 with an honest quality label. One tap to recall. No protocol knowledge required.
Group modes
Sonos-only → native Sonos API → 24-bit/48kHz lossless Bluesound-only → BluOS API → 24-bit/192kHz lossless Mixed / whole house → AirPlay 2 → 16-bit/44.1kHz lossless CD

Audio Quality Positioning
Honest, clear, cuts through the marketing noise — a Sorriva differentiator
The honest position — what Sorriva delivers
Positioning
Sorriva delivers lossless audio on every path. The only exceptions are Spotify (not lossless at the source — a Spotify limitation, not Sorriva's) and mixed-hardware AirPlay 2 groups (still lossless CD quality, not lossy). No user ever gets lossy audio from Sorriva unless their streaming service is Spotify.
By endpoint
Bluesound → 24-bit/192kHz lossless hi-res Sonos → 24-bit/48kHz lossless AirPlay 2 only → 16-bit/44.1kHz lossless CD iPhone/iPad direct → full native resolution
Consumer education — lossless vs hi-res
Positioning
The market conflates lossless and hi-res constantly. Sorriva's in-app education cuts through this: lossless means nothing is thrown away (FLAC, ALAC — bit-perfect to the master). Hi-res means above CD resolution (24-bit, 96kHz+). You can have lossless at CD quality — which is excellent — or lossless hi-res. Spotify gives you neither. Apple Music and Qobuz give you lossless. Qobuz on Bluesound gives you lossless hi-res. The Signal Path indicator makes the distinction visible at all times.
MQA — not building
Not Building
MQA Ltd went into administration in 2023. Tidal — the only major service that pushed it — has been walking it back in favor of standard FLAC. Qobuz never supported it. Building MQA decoding in 2026 would add complexity for a format the industry has abandoned. Sorriva supports FLAC natively — the standard the industry is converging on.
Per-device audio configuration — not necessary
Handled automatically
Roon requires per-device setup for MQA capabilities, max PCM bit depth, and multichannel downmix behavior — because RAAT pushes audio from a server and needs to know each device's ceiling before sending. Sorriva's architecture makes this screen unnecessary. The delivery protocol and the device negotiate quality directly: Qobuz Connect tells the Bluesound Node what's available, the Node pulls at its native ceiling. Sonos caps itself at 24/48 regardless. AirPlay 2 caps at 16/44.1 regardless. No user configuration, no device setup screens, no ceiling mismatches. The device knows what it can handle. Sorriva gets out of the way.
Roon settings Sorriva doesn't need
MQA capabilities — not building (MQA is dead) Max bits per sample (PCM) — device and protocol handle automatically Multichannel mixing / downmix — out of scope, source material is stereo Volume control mode — device native volume always used
