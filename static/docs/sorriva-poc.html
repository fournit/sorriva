Pre-session checklist

These must be done before the session starts. None take more than a few minutes, but any one of them blocked mid-session costs an hour.

  • !
    Enable Sonos Local API S2 app → Settings → System → Advanced → Local API. Toggle it on. Without this, zone discovery returns nothing and the entire session is blocked.
  • Provision Sorriva backend in Hub Run create-app.sh, port TBD, sorriva_db, LaunchDaemon. Follow standard Passione provisioning pattern. Logs under Hub sessions.
  • Create Xcode project SwiftUI, iOS target, Passione brand applied. Even an empty project started before session avoids first-boot Xcode delays.
  • Confirm MusicKit entitlement Not needed for this POC (iHeart only), but verify the Apple Developer account has the entitlement available for Phase 3. No action needed now if confirmed.
  • Know your iHeart station IDs Pick 5–6 stations you actually listen to. Browse iheart.com — the station ID is in the URL (iheart.com/live/2241/). Note them down. Used as fallback if search is slow to build.

POC goal

Ship a SwiftUI app you'd open every day at home. Pick a Sonos zone (or group several), search or browse iHeart stations, star your favorites, tap to play. Volume and transport from the Now Playing strip. Favorites persist in Postgres — they follow you across reinstalls and future devices.

Why this scope is right. iHeart station names and logos are fully available from the unofficial v2 API — no partnership needed. The Flask backend resolves stream URLs server-side and hands them to Sonos via loadStreamUrl. There is no novel architecture here. Every pattern is already proven in the Passione portfolio.
Sonos Radio is out of scope. Sonos Radio is a Sonos cloud service not exposed through the local API. iHeart streams are just HLS/MP3 URLs — the integration is clean and local-first. Spotify Connect comes in Phase 2 once the zone infrastructure exists.

Flask backend routes

Route Purpose
GET/api/zones Polls Sonos local API. Returns zone list with name, ID, group state, current transport state.
POST/api/zones/group Joins a list of speaker IDs into a group. Body: {"coordinator": id, "members": [id, ...]}
POST/api/zones/ungroup Removes a speaker from its current group. Body: {"zone_id": id}
GET/api/iheart/search Proxies to iHeart v2 search. Query param q=. Returns array of {id, name, logo_url, description}.
GET/api/iheart/station/{id} Resolves HLS stream URL for a given station ID. Returns {stream_url, name, logo_url}.
POST/api/zones/{id}/play Sends stream URL to Sonos zone via loadStreamUrl. Body: {"stream_url": "..."}
POST/api/zones/{id}/pause Pauses playback on the zone.
POST/api/zones/{id}/stop Stops playback on the zone.
POST/api/zones/{id}/volume Sets volume. Body: {"level": 0–100}
GET/api/favorites Returns user's starred stations ordered by created_at DESC.
POST/api/favorites Stars a station. Body: {"station_id", "station_name", "logo_url"}. Upserts on conflict.
DEL/api/favorites/{station_id} Removes station from favorites.

iHeart API notes

Unofficial API. iHeartRadio's official developer program is defunct. The v2 API is the internal API used by iHeart's own web and mobile apps. It has been stable for years and is used by home automation platforms, Logitech Media Server plugins, and others. Risk: it could break without warning. For a personal home tool this is acceptable — if it changes, update the endpoint.

Key endpoints

EndpointReturns
us.api.iheart.com/api/v2/content/liveStations/{id} Full station object: name, logo, streams.hls_stream, streams.shoutcast_stream, description, call letters.
us.api.iheart.com/api/v2/content/liveStations?keywords={q} Search results array. Same station object shape. Use hits array from response.

Stream URL strategy

Prefer streams.secure_hls_stream if present, fall back to streams.hls_stream, then streams.shoutcast_stream. Sonos handles all three via loadStreamUrl. Resolution happens server-side in Flask — the iOS app never calls iHeart directly.

Logo URLs

Logo URLs from the API are hosted at i.iheart.com/v3/re/new_assets/.... They load reliably. Cache them in the favorites table so the UI works if iHeart is unreachable.

Sonos local API notes

Local API must be enabled. S2 app → Settings → System → Advanced → Local API. This is the single most important pre-session step. Without it the API returns nothing.

The Sonos local API runs on port 1400 on each speaker. Discovery is via UPnP/SSDP — the Flask backend scans the local network, finds all Sonos devices, and builds the zone topology from the coordinator relationships.

Recommended library

Use soco (SoCo) — the canonical Python Sonos library. Well maintained, handles zone topology, grouping, and loadStreamUrl cleanly. Install in the shared venv: pip install soco --break-system-packages.

Grouping model

Every Sonos group has one coordinator and zero or more members. To play to a group, send commands to the coordinator. To group speakers, call zone.join(coordinator). To ungroup, call zone.unjoin().
