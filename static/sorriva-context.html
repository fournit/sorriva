Project Overview

What It Is

  • Unified music graph across local library + every streaming service
  • FLAC at home, Apple Music / Spotify away — seamlessly
  • AirPlay 2 multi-room with saved zone groups
  • AI discovery via Deriva across your full music world
  • Pure App Store product — nothing to install at home

What It Is Not

  • A self-hosted server (no Mac Mini required)
  • A Spotify competitor (you keep your subscription)
  • A hardware company (no Sorriva speakers)
  • A Roon clone (simpler, cheaper, no server)
  • Android (AirPlay 2 is Apple-only — intentional)

The gap nobody fills

Spotify owns the catalog. Sonos owns the hardware. Roon owns the audiophile server. Nobody owns the intelligent layer that unifies all of them into one coherent, beautiful experience. That's Sorriva.

Unified Music Graph

Every track in the user's world exists once in Sorriva's understanding. Source is just an attribute resolved at playback time. Sorriva owns the playlist, the history, the discovery, the identity.

Track: Kind of Blue — Miles Davis — So What
  └── Sources:
        ├── Local library (FLAC, 24-bit)     ← home, preferred
        ├── Apple Music                       ← away, lossless
        └── Spotify                           ← away, fallback
  └── Sorriva playlists: ["Late Night Jazz", "Favorites"]
  └── Play count: 12 · Last played: Tuesday
  └── MusicBrainz ID: [canonical cross-platform identifier]

Context-Aware Source Resolution

Playback request for a track
        │
        ▼
On home network?
        ├── YES → local library (FLAC/ALAC preferred)
        │           fallback: Apple Music → Spotify
        └── NO  → Apple Music → Spotify
                  (streaming services only — no local library

User never thinks about this. It just plays.

Track Identity Matching

MusicBrainz IDs (MBID) as canonical cross-platform track identifiers. Open matching databases bridge Spotify IDs, Apple Music IDs, and local files. 90%+ match rate on typical libraries. Graceful fallback for live versions, remasters, and regional variants.

Multi-Room & Zones

Sorriva groups zones using native device APIs — not AirPlay 2. Sonos zones are grouped via the Sonos HTTP API. Bluesound zones via the BluOS API. Each ecosystem handles its own sync at firmware level — the same approach Roon uses. AirPlay 2 is reserved as a fallback for cross-ecosystem groups.

Zone Grouping Model

Sonos-only group (e.g. "Downstairs")
  → Sorriva calls Sonos HTTP API
  → Sonos handles sync natively
  → Quality: 24-bit/48kHz lossless

Bluesound-only group (e.g. "Listening Room")
  → Sorriva calls BluOS API
  → BluOS handles sync natively
  → Quality: 24-bit/192kHz lossless

Mixed group (e.g. "Whole House" — Sonos + Bluesound)
  → AirPlay 2 fallback — only cross-ecosystem option
  → Quality: 16-bit/44.1kHz lossless CD
  → Honestly labelled in UI — user sees quality badge before pressing play

Sorriva advantage over Roon

Roon cannot group across ecosystems at any quality level — a Sonos zone and a Bluesound zone simply cannot play together in Roon. Sorriva solves both ends: best quality within ecosystems, everywhere via AirPlay 2 fallback. One decision, clearly labelled. No protocol knowledge required.

Saved Zone Groups

User creates named groups — "Downstairs", "Party Mode", "Whole House" — once, recalls with one tap. Sorriva automatically selects the best delivery path for the hardware mix. Quality badge shown on every group before playback begins.

Compatible Hardware Ecosystem

Works with any device supporting AirPlay 2, Spotify Connect, Qobuz Connect, or Tidal Connect — essentially all serious audio hardware made in the last five years:

Sonos Bluesound Bang & Olufsen Bowers & Wilkins Devialet KEF Naim Marantz Denon Yamaha NAD WiiM

Audio Quality

Sorriva delivers lossless audio on every path. Nobody gets lossy audio except via Spotify (source limitation, not Sorriva's). Hi-res above CD quality requires Bluesound hardware. The Signal Path Indicator makes this visible at all times.

Key Concepts

Lossless means the audio data is mathematically identical to the original — nothing thrown away during compression. FLAC and ALAC are lossless. This is the line that matters most.

Hi-res means above CD resolution (24-bit, 96kHz+). Can also be lossless. Whether humans can reliably distinguish hi-res from lossless CD quality in blind tests is genuinely contested. Sorriva supports it where the architecture allows and is honest where it doesn't.

Lossless CD quality (16/44.1) is excellent audio. It already exceeds human perceptual limits. Not a compromise — just not hi-res.

Non-Features (Architecture makes them unnecessary)

  • MQA support — not building. MQA Ltd in administration 2023. Industry standardizing on FLAC.
  • Per-device PCM bit depth config — unnecessary. Device and delivery protocol negotiate automatically. Only needed in Roon because RAAT pushes audio from a server.
  • Multichannel downmix config — out of scope. Local libraries and streaming services are overwhelmingly stereo.

Device Control · EQService

Audio delivery and device control are two separate, simultaneously running layers. Connect protocols handle audio delivery. Native device APIs handle configuration and EQ.

Sorriva
  ├── Audio delivery layer
  │     ├── Qobuz Connect / Tidal Connect  → Bluesound (24/192)
  │     ├── Sonos HTTP API                 → Sonos zones (24/48)
  │     ├── Spotify Connect                → any Connect device
  │     └── AirPlay 2 (fallback)           → cross-ecosystem groups, AirPlay-only devices
  │
  └── EQService layer     → configuration, EQ, tone controls
        ├── Sonos HTTP API   → bass, treble, loudness, balance, Trueplay
        ├── BluOS HTTP API   → EQ, tone controls, device config
        └── Plain AirPlay 2  → volume only (HomePod, etc.)

Abstraction Principle

One Sorriva EQ UI. EQService translates to each device's native API behind the scenes. Users see one consistent interface regardless of device. New device type = write one driver, UI never changes. Devices without EQ APIs show a reduced panel with an honest explanation — not broken, just honest.

Apple Ecosystem Only — Intentional

AirPlay 2 is the core multi-room differentiator. Android has no AirPlay 2 support. The Android equivalent (Google Cast) has a weaker device ecosystem and would require a parallel architecture — essentially a separate product. Being Apple-first is a deliberate product decision, not an apology. Windows users are fully supported — the app lives on Apple devices; the user's PC is invisible to Sorriva.

Marketing Positioning

The Core Message

Sorriva is the music app for people who care about their system — but are tired of being locked into one ecosystem's idea of how it should work.

Hardware Freedom — Key Differentiator

The two dominant players both use hardware lock-in as a business model. Sonos requires Sonos hardware. Roon requires Roon Ready endpoints. Sorriva's answer is the opposite:

If it supports AirPlay 2, Spotify Connect, Qobuz Connect, or Tidal Connect, it works. A $79 WiiM Mini, a $1,000 Marantz M1, a $3,500 Naim Mu-so, a Bluesound Node into a NAD preamp — user's choice, user's budget, user's taste. No Sorriva hardware to buy. No certification program to join. No Roon Ready required.

Audio Quality — Consumer Education Positioning

The market conflates lossless and hi-res constantly. Sorriva cuts through it with an honest, clear position:

  • Nobody gets lossy audio in Sorriva — except via Spotify, which isn't lossless at the source. That's Spotify's limitation, not Sorriva's, and the Signal Path Indicator shows it clearly.
  • Lossless CD quality (16/44.1) is delivered on every path including mixed-hardware AirPlay 2 groups. Excellent audio — not a compromise.
  • 24-bit/48kHz lossless delivered to Sonos hardware via native Sonos API. Same ceiling as using the Sonos app directly.
  • 24-bit/192kHz lossless hi-res delivered to Bluesound hardware via Qobuz Connect and Tidal Connect. Full resolution, device pulls directly from cloud.
  • Hi-res requires the right hardware. Honest, clear, set before subscription.
